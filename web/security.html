Infrastructure & Hosting

Where is PrismRAG hosted?

PrismRAG runs on Microsoft Azure Container Apps (US East by default). Enterprise plans can request EU West (Ireland) or AP Southeast (Singapore) data residency — contact sales to enable a regional deployment for your organization.

What uptime SLA do you offer?

We target 99.9% monthly uptime for Professional and Enterprise plans. Current and historical status is available at /status. The SLA with credit terms is defined in our SLA page.

Is the infrastructure shared or dedicated?

All plans use shared compute infrastructure (multi-tenant). Database rows are isolated by tenant_id with Row-Level Security enforced at the Postgres layer. Enterprise plans can request a dedicated namespace — contact sales.

Data & Encryption

How is data encrypted?

All data is encrypted in transit with TLS 1.3. Data at rest (Postgres, vector embeddings, model weights) is encrypted with AES-256 via Azure managed keys. Enterprise customers can supply their own Azure Key Vault key (CMEK) — contact sales to configure.

Who can access my mapping data and embeddings?

Your data is strictly isolated by tenant. Insight IT Solutions staff cannot query your tenant data without explicit written consent. Access is logged in an immutable audit trail. We never train our foundation models on your proprietary data.

Do you process personal data (PII)?

PrismRAG is designed to process domain vocabulary, mapping rules, and semantic vectors — not PII. If your use case involves PII (e.g., patient names in a healthcare mapping), execute a Data Processing Agreement (DPA) before ingesting. Our standard DPA is available for download and execution.

Can I delete my data?

Yes. Deleting a tenant via the API permanently removes all mapping versions, chunk embeddings, graph data, and model weights within 72 hours. For a certified deletion certificate (required by some compliance frameworks), contact privacy@insightits.com.

Authentication & Access Control

What authentication methods are supported?

Email + password (bcrypt, PBKDF2 fallback) — all plans.
API keys (prk_… prefix, SHA-256 hashed in DB) — all plans.
SSO / OIDC (Azure AD, Okta, Google Workspace, any OIDC-compliant IdP) — Professional and Enterprise.
SCIM 2.0 provisioning (RFC 7644) — Enterprise only.

Is MFA available?

Yes. TOTP-based MFA (RFC 6238, compatible with Google Authenticator, Authy, 1Password) is available for all plans. Enterprise organizations can enforce MFA org-wide — any user without MFA enabled will be blocked from API access until they enrol at /api/v1/mfa/enrol.

Can I restrict API access to specific IP addresses?

Yes — Enterprise feature. Org admins can define a CIDR allowlist via PUT /api/v1/org/{org_id}/security/ip-allowlist. When enabled, any request from an IP outside the allowlist is rejected with HTTP 403 before authentication runs.

What rate limits apply?

Every API response includes X-RateLimit-Limit, X-RateLimit-Remaining, and X-RateLimit-Reset headers so your client can implement back-off proactively.

Compliance & Certifications

Are you SOC 2 certified?

We are currently undergoing SOC 2 Type II audit (target completion Q4 2026). In the meantime, we provide a security questionnaire, architecture overview, and reference architecture document upon request under NDA.

Are you GDPR / CCPA compliant?

Yes. We act as a Data Processor under GDPR and CCPA. Our DPA covers lawful basis, data subject rights (access, erasure, portability), breach notification (within 48 hours), and subprocessor obligations. EU customers can request Standard Contractual Clauses (SCCs) by emailing privacy@insightits.com.

Have you had an independent penetration test?

A penetration test is scheduled for Q3 2026 with an accredited third-party firm. Report summaries will be available to Enterprise customers under NDA upon completion.

Do you have a bug bounty program?

We operate a responsible disclosure program. If you discover a vulnerability, email security@insightits.com with details. We target a 48-hour acknowledgement and 30-day remediation window for critical issues. A formal bug bounty program is planned for H2 2026.
